This starter has a versioning system built in. It lets you create versions of your prototype and imports the routes for each one.

How the folders are structured

You can create version folders, for example v1, at 3 different levels, depending on how you like to structure your prototypes.

Example level structures:

  1. /v1
  2. /juggling-licence/v1
  3. /juggling-licence/new-feature/v1

There is an example-feature folder within views. In there is an index page which acts as the changelog for each feature. There is also a v2 folder which has an example question page and a _routes.js file.
